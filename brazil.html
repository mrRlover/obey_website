3 Data Preps

In this section, hte raw data is transformed into data neeeded for the tests. The main transformations are calculation percentage change and matching dates with temperature data dates. The temperature data has daily data. Share price data misses holidays and weekends. Search data is weekly. These series need to have data every day like the temperature series. This is done by joining on the Dates in temperature, then filling the NAs of the series. Price data is filled last observation carried forward. The rest are filled backwards.

To get change following the log formula:

\[ R_{it} = ln\frac{P_{it}}{P_{it-1}} \] Using the log rules, this is equivalent to:

\[ln(P_{it}) - ln(P_{it}) \] Which in R, is implemented as diff(log(x)), as seen below.

br_temp = br_temp %>% 
  mutate(DATE = ymd(DATE)) %>%
  mutate(TEMP = c(NA, diff(log(TEMP)))) %>% # log change
  slice(-1)

Days are now daily.

The same logic is repeated for the rest, only is backfilled, by setting the fromLast argument of na.locf to TRUE.

The join to create daily observations creates NAs on dates that were not previously there. Once there are daily dates, there is need to backfill the values for that week. So first create a variable that can group observations by week. Once that is done, simply check that each group is a full week by checking if the number of rows per group (n()) is less than 6, if it is, then leave the NA values as is, else backfill the NA values. Note that there can only be less than 6 days if it’s in the beginning or at the end, i.e. series ends and/or starts in the middle of a week.

br_policy = br_policy %>% 
  mutate(DATE = ymd(Date)) %>% 
  right_join(br_temp) %>% 
  arrange(DATE) %>%
  select(-Date) %>% 
  mutate(Week = paste0(year(DATE), week(DATE))) %>% 
  group_by(Week) %>% 
  mutate(climate_change = ifelse(n() < 6, `climate change: (Brazil)`,
        na.locf(`climate change: (Brazil)`, fromLast = T))) %>% 
  ungroup() %>% 
  mutate(POLICY = c(NA, diff(log(climate_change)))) %>% 
  mutate(POLICY = case_when(is.na(POLICY) ~ 0, is.nan(POLICY) ~ 0,
                            is.infinite(POLICY) ~ 0,
                            TRUE ~ POLICY))

3.2 Market cap weighted

Portfolio return using market cap as weights. Here, market cap is proxied by volume. The individual share weight is share volume divided by total volume

w = br_volume %>% 
  mutate(DATE = as.Date(Date, origin = "1899-12-31")) %>% 
  select(DATE, everything()) %>% 
  right_join(br_temp) %>% 
  arrange(DATE) %>% 
  select(-c(Date, TEMP)) %>%
  select(DATE, everything()) %>% 
  slice(-1)

w[, -1] = w[, -1] / rowSums(w[, -1], na.rm = T)

The portfolio return is the average of the return multiplied by the weights.

br_mret_cap = rowMeans(br_ret[, -1] * w[, -1], na.rm = T) %>% 
  as.data.frame() %>% 
  mutate(DATE = br_ret$DATE) %>% 
  select(DATE, everything()) %>% 
  `colnames<-`(c("Date", "Mean"))

The graph above shows that the coefficients are not constant through time. The policy coefficients hover around 0, temperature changes are slightly larger, going above and below the OLS estimate by a magnitude of around 0.05. Interestingly, the coefficients start out high at 0.25, then dramatically dropping to zero in a short space of time. The technology coefficients experiences dramatic swings but has a lot of missing data, where presumably, coefficients couldn’t be estimated, possibly due to the nature of the data.

5.3 Model Diagnostics

The linear model from base R has an internal method for diagnostic plots, so this is simply a matter of calling plot() on the model.

par(mfrow = c(2, 2))
plot(eq1)

par(mfrow = c(2, 2))
plot(eq1_cap)

R does not have a function to execute time varying regressions, this comes via the external package, tvReg. However, the package does not include any post regression statistical testing or model diagnostics. As a result, the plot() does not have a method for regressions of tvLM(). Therefore, this has to be done manually.

First the residuals and fitted values are extracted.

tvEQ1_res <- data.frame(x = seq_along(tvEQ1$residuals), 
                        y = tvEQ1$residuals)

tvEQ1_fit <- data.frame(x = tvEQ1$fitted, y = tvEQ1$residuals)

Then using ggplot, the graphs are created.

normqq <- ggplot(tvEQ1_res, aes(sample = y)) + 
  stat_qq() + 
  stat_qq_line()+
  ggtitle("Normal Q-Q")+
  theme_bw()+
  theme(panel.grid.major = element_blank(),
        panel.grid.minor = element_blank())

resid_fit <- ggplot(tvEQ1_fit, aes(x, y))+
  geom_point()+
  geom_smooth(se = F)+
  ggtitle("Residuals vs Fitted")+
  theme_bw()+
  theme(panel.grid.major = element_blank(),
        panel.grid.minor = element_blank())

scale_loc <- ggplot(tvEQ1_fit, aes(x, sqrt(abs(y))))+
  geom_point(na.rm=TRUE)+
  stat_smooth(method="loess", na.rm = TRUE, color = "red", se = F)+
  xlab("Fitted Value")+
  ylab(expression(sqrt("|Standardized residuals|")))+
  ggtitle("Scale-Location")+
  theme_bw()+
  theme(panel.grid.major = element_blank(),
        panel.grid.minor = element_blank())

Using grid.arrange to put the graphs in one plot

grid.arrange(resid_fit, normqq, scale_loc, ncol=3)
